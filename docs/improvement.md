# 可继续改进的方向

当前实现突出「流程清晰 + 可解释 + 可展示」，在方法深度与实验严谨性上刻意保持克制。以下列出了**最值得优先做的几条延伸**（按投入产出比大致排序），便于写入报告「Future Work」或作为第二阶段的实现目标。

---

## 1. 数据与任务：更贴近 Web Usage Mining

- **会话（session）建模**：将 MovieLens 按时间戳聚类为浏览会话（如间隔超过 30 分钟切分会话），在 Recent 视角用「当前会话内的行为」而非固定 `k` 条，叙事上更接近真实点击流。
- **负采样与评估**：BPR 使用均匀负采样会偏乐观；可改为 **Popular negative** 或 **hard negative**（与正样本同 genre/popularity 桶），测试集评估与训练假设更一致。
- **冷启动 / 长尾**：单独报告「低活用户 / 长尾物品」子集上的 Recall@K，体现方法在稀疏情形下的行为。

---

## 2. 多视角图：从「手工规则」到「可学习或更丰富先验」

- **时间衰减边权**：Global 视角对边赋权 \(w_{ui} \propto \exp(-\lambda \Delta t)\)，再按权构造归一化邻接（需确认 LightGCN 对加权二部图的传播形式，或退化为多层时间窗子图）。
- **物品内容 / 侧信息**：若有 genre，可构建「同体裁共现」辅助图做 **multi-graph fusion**（简单版：额外一张图共享同一套 user/item embedding 做另一路传播后加权求和）。
- **视角缺失时的鲁棒性**：部分用户 Recent 极短时的 **fallback**（回退到 Global 或插值两种视角的 embedding），并在 demo 里展示。

---

## 3. 对比学习：当前 InfoNCE 较简陋

- **多正样本 / 队列**：用 **MoCo 式 memory bank** 或更大 batch 稳定对比学习，减轻 batch 内假负样本噪声。
- **视角对的选择**：不仅 Global–Recent，可轮换 Global–Frequency 或三视角 **cycle consistency**（链式对齐），并做 \(\lambda_{cl}\) 与温度 \(\tau\) 的敏感性曲线（1–2 张图即可）。
- **与主任务解耦**：对比损失仅加到若干层或 **stop-gradient** 一路，减少对 BPR 梯度的干扰（常能缓解「加了 CL 反而掉点」）。

---

## 4. 模型：在保持可讲清的前提下加深一层

- **轻量化图注意力**：在 user–item 消息传递上用 **GAT 一层 + LightGCN 若干层** 混合，强调「近期邻居权重更高」，仍可控在课程项目规模。
- **因果 / 去偏叙事（展示向）**：引入简单的 **IPS 加权** 或 popularity-bias 校正项，在报告里讨论「曝光偏差」——不必完整实现因果推荐，一条 baseline 对比即有亮点。

---

## 5. 工程与可复现

- **超参与早停**：验证集 Recall@K + patience，避免固定 80 epoch；记录 `config` + git commit + 随机种子到一次 `runs/` 日志。
- **更大或更接近 Web 的数据**：MovieLens 1M / MovieLens-25M（注意内存与评估时间）；或 **Yelp / Amazon** 评分的隐式化，增强「规模感」。
- **推理加速**：训练后导出用户/物品向量，Top-K 用 **ANN（faiss）** 或 `numpy` 分区，demo 交互延迟更低。

---

## 6. 展示与论文叙事（低成本高回报）

- **Attention rollout 或邻居重要性**：若引入 GAT，可视化「推荐给某用户时，贡献最大的若干邻居物品」，比纯 t-SNE 更直观。
- **统计显著性**：固定多种子重复 3–5 次，报告均值±方差或置信区间，让「完整模型 vs baseline」对比更有说服力。

---

挑选 **1（会话 + 评估）+ 3（CL 调优或小改造）+ 6（可视化或重复实验）** 通常能在 1–2 周内把项目从「基础课设」提升到「有明确增量与讨论深度」的档次，且仍适合暑期项目展示节奏。
